# codeguard v0.2 — Context-Aware Review Engine

## Goal

Transform codeguard from a simple regex pattern matcher into a context-aware code reviewer that dramatically reduces false positives and adds high-value detection categories from TrailOfBits patterns.

## What Changes

### 1. Context Layer (new module: `src/context/`)

A lightweight analyzer that runs before rules, providing context about each line:

**Line-level context:**
- `isComment` — line is inside a comment (single-line or block)
- `isString` — line is inside a string literal or template literal
- `isImport` — line is an import/require statement

**Region-level context:**
- `functionBoundaries` — start/end line of each function
- `classBoundaries` — start/end line of each class
- `tryCatchRegions` — lines inside try/catch blocks

**File-level context:**
- `fileType` — test, config, utility, migration, generated
- `imports` — what the file imports (detect logger utilities, test frameworks)

**Implementation:** Regex-based single-pass line scanner. No AST, no dependencies. Tracks bracket depth for regions. Detects comments via `//`, `/* */`, `#`, `"""`. Detects strings via quote tracking with escape handling.

### 2. Rules Get Context

Change rule signature from:
```typescript
detect(lines: string[], filePath: string): Finding[]
```
To:
```typescript
detect(lines: string[], filePath: string, context: FileContext): Finding[]
```

Rules use context to:
- Skip findings on comment/string lines (all rules)
- Skip findings in test/config files (security rules)
- Adjust confidence based on location (inside try/catch = higher for error rules)
- Check imports (has logger? has sanitizer?)

### 3. New Rule Categories

**Insecure Defaults (from TrailOfBits insecure-defaults):**
- `SEC006` fail-open-env: `env.get(X) or 'default-secret'` patterns
- `SEC007` debug-flag-default: `DEBUG = true` or `debug: true` in config
- `SEC008` permissive-cors: `Access-Control-Allow-Origin: *` or CORS allow-all
- `SEC009` weak-crypto: MD5, SHA1, DES, RC4, ECB in security contexts

**Sharp Edges (from TrailOfBits sharp-edges):**
- `SEC010` unsafe-deserialization: Unsafe deserialization patterns (loading untrusted serialized data without safe loaders)

**Error Handling (expanded from silent-failure-hunter):**
- `ERR005` async-no-try-catch: async function without error handling
- `ERR006` optional-chain-critical: optional chaining on security/auth paths

**Code Quality (expanded):**
- `QA006` unused-import: import not referenced in file body
- `QA007` duplicate-condition: same condition checked twice in if/else chain

### 4. Multi-line Pattern Support

Add `detectMultiline(content: string, filePath: string, context: FileContext): Finding[]` to Rule interface (optional). For patterns that span lines like multi-line empty catch blocks.

Current rules already handle some multi-line via line-by-line lookahead. New method allows full-content regex for complex patterns.

## What Stays The Same

- Zero external dependencies
- Same MCP tools (review_file, review_diff, check_error_handling, list_rules)
- Same Finding/ReviewResult types
- Same severity/confidence scoring
- Same project structure conventions (matching orchestr8)

## Architecture After v0.2

```
file content
    |
    v
[Context Analyzer] -- single pass, builds FileContext
    |
    v
[Rule Engine] -- passes FileContext to each rule
    |
    v
[Findings] -- fewer false positives, richer categories
```

## New File Structure

```
src/
  context/
    types.ts          # FileContext, LineContext, RegionInfo
    analyzer.ts       # buildFileContext() - single pass scanner
    index.ts          # barrel export
  rules/
    insecure-defaults.ts  # SEC006-010 (new)
    error-handling.ts     # ERR001-006 (expanded)
    security.ts           # SEC001-005 (existing, now context-aware)
    code-quality.ts       # QA001-007 (expanded)
    engine.ts             # Updated to pass context
```

## Success Criteria

- False positives reduced: no flags inside comments, strings, test fixtures
- ~25-30 total rules (up from 14)
- All existing tests still pass
- New tests for context layer and new rules
- Zero new dependencies
